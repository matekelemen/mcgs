◆ maxStallCount

template<class TValue >
int mcgs::ColorSettings< TValue >::maxStallCount

Controls how many consequent failed coloring iterations to allow before terminating.

Since the algorithm randomly picks colors for each vertex from their remaining palettes, it is possible to end up with iterations that fail to assign valid colors to any of the remaining vertices. This is characteristic of the final stages of the coloring process for runs with too tight initial palettes. Allowing a longer chain of failed iterations makes the algorithm more robust in these cases.

Note
  • Negative values for maxStallCount allow infinite stalling.
  • Defaults to 1e3.

◆ shrinkingFactor

template<class TValue >
int mcgs::ColorSettings< TValue >::shrinkingFactor

Parameter controlling the initial palette size.

This is the only algorithmic parameter, but it has a significant influence on the quality of the resulting coloring as well as the runtime of the algorithm. In general, a shrinking factor that leads to a good coloring also has shorter runtimes.

The initial palette size \(p\) depends on the maximum vertex degree \(v_{max}\) and shrinking factor \(s\):

\[ p = \frac{v_{max}}{s} \]

Note
  • If a nonpositive value is passed, the minimum vertex degree is used instead.
  • Defaults to -1.

◆ tolerance

template<class TValue >
TValue mcgs::ColorSettings< TValue >::tolerance

Minimum absolute value for matrix entries to treat as nonzero.

Some software store actual zeros in their sparse matrices. For example, this can happen during the naive enforcement of Dirichlet conditions or linear constraints in FEM software. Topology complexity can be reduced in these cases by defining a small tolerance, under which entries in the sparse matrix are ignored.

Note
  • Nonpositive values for tolerance will respect the matrix topology as is.
  • Defaults to 0.

◆ verbosity

template<class TValue >
unsigned short mcgs::ColorSettings< TValue >::verbosity

Verbosity level controlling the volume of output to std::cerr and std::cout.

Each level is a strict superset of lower levels.

  • 0: no output is written to std::cerr or std::cout.
  • 1: error messages are written to std::cerr.
  • 2: information outside iterations is written to std::cout.
  • 3: information from iterations is written to std::cout.
Note
Defaults to 1.
